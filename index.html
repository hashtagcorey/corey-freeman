Calculate The Third Thursday of the Month (PHP)

This code automatically calculates the 3rd Thursday of the month, and determines if it is closer in time to the current month. The result is either the third Thursday of the current month, or the third Thursday upcoming, whichever is next. This allows for an automated alert for the upcoming showcase, removing a tedious recurring administrative task.


//Calculate Show Dates for Overload
function overload_dates() {
$last_show = date("M-d-Y", strtotime("third thursday of this month")); //christmas day after
$next_show = date("M-d-Y", strtotime("third thursday of next month")); //
//Last Show Sooner than Next? Check again in january.
if($last_show >= $next_show) { $display_show = $last_show; } 
if($last_show <= $next_show) { $display_show = $next_show; }
return $display_show;
}

Create an Archive Page via Meta Data (PHP w/ WordPress)

A work-around for creating individual archive displays, based on a custom taxonomy in WordPress. Instead of listing the content by type or author, each "page" filed under a certain "term" is organized and named and the pages are individually linked to. This results in dynamic archive generation that can be shown and linked anywhere on the website. Escapes have been added where necessary to display source code.

function show_all_comics() {
$args = array( 'hide_empty=0' );
$terms = get_terms( 'story', $args );
if ( ! empty( $terms ) && ! is_wp_error( $terms ) ) {
    $count = count( $terms );
    $i = 0;
    $term_list = '<\div class="rfd-comic-archive"><\h4>Stories:<\/h4>';
    foreach ( $terms as $term ) {
        $i++;
        $term_list .= '<\a class="rfd-comic-archive-link" href="' . esc_url( get_term_link( $term ) ) . '" alt="' . esc_attr( sprintf( __( 'View all post filed under %s', 'my_localization_domain' ), $term->name ) ) . '">' . $term->name . '<\/a>';
        if ( $count != $i ) {
            $term_list .= ' · ';
        }
        else {
            $term_list .= '<\/div>';
        }
    }
 echo $term_list;}}

Highlight Menu Item On Scroll (jQuery)

Adapted for a one-page website, I tweaked the code to work automatically when a WordPress block group is present. It allows for the creation of scrolling sections without individually coding a ton of anchors. Because of this, adding and removing page sections is scalable.


jQuery(function ($) {
    //Set first menu item as active
    $('.menu-item:first-child a').addClass('rfd-menu-active')
    $(window).scroll(function () {
        //Create variable to get the current vertical position
        var scrollPos = $(window).scrollTop();
        //For each section with the class ds-section
        $('.wp-block-group').each(function (i) {
            //Create variable to calculate when to add active class
            var topPos = $(this).offset().top;
            //If the section is equal to or less than 80px from the top of the page
            if ((topPos - scrollPos) <= 200) {
                //Remove the active class from the previous menu item
                $('.rfd-menu-active').removeClass('rfd-menu-active')
                //Add the class to the current menu item
                $('.menu-item a').eq(i).addClass('rfd-menu-active')
            }
        })
    });
});
